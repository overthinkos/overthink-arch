# Changelog — 2026-06

### 2026-06-28 — build: bake plugin-feature into /usr/lib/charly/plugins (feature externalization)

The superproject's feature cutover externalized `charly feature` (the ADE inspection verbs
list/pending/validate) out of charly's core into `candy/plugin-feature` (`command:feature`). The
per-plugin bake loop now builds + installs plugin-feature beside plugin-secrets/udev/tmux/preempt, so the
INSTALLED host charly resolves + `syscall.Exec`s `charly feature` project-lessly (it shell-backs to the
hidden core `__feature-list`/`__feature-pending`/`__feature-validate` commands invoking the in-core
loader + plan/Step model + validatePlanSteps — which stay core).

R10 (fully tested and validated): check-feature-local PASS run 2026.179.0423.

### 2026-06-28 — build: bake plugin-udev into /usr/lib/charly/plugins (udev externalization)

The superproject's udev cutover externalized `charly udev` (the GPU-device udev-rule manager) out
of charly's core into `candy/plugin-udev` (`command:udev`). The package now ships that plugin beside
the charly binary so the INSTALLED host charly resolves + `syscall.Exec`s `charly udev` project-lessly

- `build()` + `package()` refactored to a per-plugin LOOP (R3) building BOTH plugin-secrets AND
  plugin-udev: each DEV path from the working tree (`${startdir}/../../candy/<name>`), AUR path from
  the clone with `GOPATH=${srcdir}/gopath` set ONLY on the AUR branch; the `.providers` words manifest
  per plugin via `charly __plugin-providers`; install the binary + manifest into `/usr/lib/charly/plugins/`.

### 2026-06-27 — build: bake plugin-secrets into /usr/lib/charly/plugins (C2 secrets externalization)

The superproject's C2 cutover externalized the credential store + the `charly secrets`
CLI out of charly's core into `candy/plugin-secrets`. The package now ships that plugin
beside the charly binary so the INSTALLED host charly resolves `verb:credential` and runs
`charly secrets` project-lessly:

- `build()` builds `candy/plugin-secrets` (GOWORK=off) — DEV path from the real working
  tree (`${startdir}/../../candy/plugin-secrets`, since makepkg clones only committed HEAD),
  AUR path from the cloned source; `GOPATH=${srcdir}/gopath` is set ONLY on the AUR path so
  the dev build leaves no read-only module cache for makepkg's cleanup to choke on. The
  `.providers` words manifest is generated from the candy's `plugin.providers` via
  `charly __plugin-providers` (single-sourced; NOT the gRPC Describe, which omits the
  CLI-served `command:secrets`).
- `package()` installs the plugin binary + `.providers` into `/usr/lib/charly/plugins/`.
- R5: the credential-surface `depends`/`optdepends` comments now state go-keyring links into
  the bundled plugin-secrets binary, not charly's core (which links no go-keyring at all).
